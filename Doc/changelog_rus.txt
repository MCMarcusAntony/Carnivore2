Chagelog for Carnifore2 MultiFunctional Cartridge
-------------------------------------------------

V14 10.02.2017

1. Регистр конфигурации расширений слота
   Mconf (+1E, 4F9E, в записи директории - 3B)
   Теперь если записывать значения типа 0xxx1111 bin (0F), то старое значение регистра менятся не будет.

2. Теперь все регистры конфигурации картриджа читаются своими значениями по месту прописки (4F80-4FA1).
   Это сделано преимущественно для удобства отладки в дебагере. Можно запретить чтение этих регистров установив
   в главном управляющем регистре CardMDR (4F80) значение нулевого бита в единицу xxxxxxx1. В таком случае
   на этом участке памяти будет читаться то что задано другими параметрами (обычно страница ROM).

V13 03.02.2017

1. Так как наш мультикартридж теперь состоит как 4 устройства каждое из которых живёт в своём расширении слота,
   то неплохо бы было ими поуправлять. с этой целью теперь существует регистр конфигурации Mconf,
   расположенный по смещению +1E относительно базы 4F80. Его битовая маска :

    7 bit = 1 - разрешение расширенного слота (включаются четыре подслота)
            0 - один слот (устройство выбраное установленным в единицу младшим битом из 3,2,1,0)
    6 bit = 1 - разрешение чтения портов RAM маппера картриджа на стандартных FC,FD,FE,FF
    5 bit = 1 - разрешение YM2413 (FM Pack Synt. 3Ch,3Dh)
    4 bit = 1 - разрешение 3С порта (МММ-маппера)
    3 bit = 1 - разрешение -3 подслота FM Pack bios ROM
    2 bit = 1 - разрешение -2 подслота 1 Mb RAM с маппером (MMM стандарт)
    1 bit = 1 - разрешение -1 подслота CF card disk интерфейс
    0 bit = 1 - разрешение -0 подслота мультикартриджа (Регистры конфигурации доступны только сдесь)
